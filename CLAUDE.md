# Narrative - Claude Instructions

## Project Overview

Narrative is a Solana-focused trading journal and analytics platform.

Target users:

* Solana traders
* Meme coin traders
* Copy traders
* Smart wallet followers

Narrative is NOT intended to be:

* A stock trading platform
* A forex trading platform
* A social network

## Product Philosophy

Narrative should feel:

* Professional
* Fast
* Clean
* Modern
* Data-driven

Avoid:

* Clutter
* Unnecessary complexity
* Placeholder content
* Unrelated features

## Development Rules

Before making any changes:

1. Read the relevant PRD.
2. Make the smallest possible change.
3. Preserve working functionality.
4. Do not rewrite unrelated code.

## Protected Areas

Do not modify without explicit approval:

* WalletImport
* QuickJournal
* Authentication
* Supabase Edge Functions

## Code Change Rules

Always:

* List files modified.
* List components modified.
* Explain why each change was made.
* Confirm what was NOT touched.

## UX Principles

Every page should answer:

1. What is this page for?
2. What should the user do next?

Empty states should:

* Explain the feature
* Suggest an action
* Never feel unfinished
